项目终端操作与模块说明（简版）

项目位置：
D:\GitHub\math-quiz-multi-agent - Copy

默认输入：
examples/input.md

默认输出：
outputs

默认流程：
Decomposer 读取原题并拆解结构
Reviewer 检查拆解是否合理
Generator 生成相似题和答案
QC 检查题目、答案、数量、风格和数学正确性
render_vscode_preview 生成 VSCode 可读预览


一、常用终端操作

进入项目目录：
cd "D:\GitHub\math-quiz-multi-agent - Copy"

完整运行项目：
python codes\run_pipeline.py

查看当前配置，不消耗 API token：
python codes\run_pipeline.py --show-options

交互式修改配置：
python codes\configure_options.py

快速默认运行：
run_default.bat

或：
python codes\run_default.py

检查代码语法：
python -m py_compile codes\validators.py codes\generator_agent.py codes\qc_agent.py codes\run_pipeline.py codes\configure_options.py

重新生成 VSCode 数学预览：
python codes\render_vscode_preview.py --output-dir outputs


二、常用可调参数

输入文件：
--input examples/input.md

输出目录：
--output-dir outputs

题目数量：
--question-count 15

题目数量模式：
--question-count-mode match_source
说明：默认按原题数量一一生成相似题。

难度：
--difficulty match_input
可改为 easy / medium / hard / 自定义描述。

知识点：
--knowledge-points "含参不等式;分类讨论"

题型：
--question-types "解不等式;分式不等式"

风格：
--style match_input

答案详略：
--answer-detail concise
可改为 detailed。

示例：
python codes\run_pipeline.py --difficulty hard --knowledge-points "含参不等式;分类讨论" --answer-detail detailed


三、配置文件

主配置文件：
config/pipeline_options.json

重要字段：
input                 输入文件
output_dir            输出目录
review_threshold      Reviewer 通过分数
qc_threshold          QC 通过分数
max_review_rounds     Decomposer 最多返工轮数
max_qc_rounds         Generator 最多返工轮数
generation_profile    出题数量、难度、知识点、题型、风格
models                各 Agent 使用的模型
thinking              各 Agent 是否开启 thinking mode
reasoning_effort      thinking 强度
token_budget          token 预警和停止限制

当前建议：
Generator thinking 开启
QC thinking 开启
question_count_mode 使用 match_source
allow_input_compaction 使用 false

五、Agent 与程序模块

codes/run_pipeline.py
总控制程序。负责按顺序运行 Decomposer、Reviewer、Generator、QC，并处理返工、最终文件、失败状态和预览生成。

codes/decomposer_agent.py
拆题 Agent。读取原始 input.md，识别题目数量、题型、知识点、难度和结构。现在会尽量识别“一道原题对应一道相似题”的 source_structure。

codes/reviewer_agent.py
审核 Agent。检查 Decomposer 的拆解是否可信，给出分数和修改建议。

codes/generator_agent.py
出题 Agent。根据原题和 Decomposer 结果生成相似题与答案。已开启 thinking mode，并加入答案自检规则，重点防止含参不等式分类错误。

codes/qc_agent.py
质检 Agent。检查生成题目和答案是否正确。会检查数量、格式、风格、数学正确性；若发现系统性答案错误，会要求整套答案重算。

codes/validators.py
本地结构校验器。不调用模型。负责检查 JSON 字段、题目数量、答案数量、题目答案是否一一对应。

codes/math_checks.py
基础数学一致性检查。不调用模型。用于拦截部分明显的因式分解和不等式错误。

codes/common.py
通用工具。负责读取 .env、调用 DeepSeek API、处理 thinking mode、记录 debug、修复 JSON、读取和写入文件。

codes/budget.py
token 预算模块。负责估算 token，超过预警值时提示，超过停止值时中断运行。

codes/configure_options.py
终端交互式配置工具。用于不用手改 JSON 的情况下调整输入、输出、模型、thinking、题目数量、难度等。

codes/render_vscode_preview.py
预览生成工具。把最终题目、答案、QC 结果整理成 outputs\vscode_preview.md，方便在 VSCode 中查看数学内容。

codes/run_default.py
快捷入口。等价于使用默认参数运行主流程。


六、常见问题

没有 DEEPSEEK_API_KEY：
在项目根目录创建 .env，并参考 .env.example 填写。

__pycache__ 是什么：
Python 自动生成的缓存目录，可忽略，不是项目核心文件。

题目数量不对：
优先查看 config/pipeline_options.json 中的 question_count_mode。
默认应为 match_source，表示按原题数量生成。

答案错得多：
Generator thinking 已开启。
如果仍然错，下一步应增加专门的数学求解或代值验证模块，而不是只继续加 prompt。

模型返回空内容：
通常是 thinking 消耗了输出预算。程序会自动关闭 thinking 重试一次。

运行太慢：
复杂题开启 thinking 后会变慢。若长期无输出，可查看 outputs/debug 和 outputs 中最新文件时间。
